## 1. Spotify App Setup

### Create a Spotify Developer App

1. Go to [Spotify Developer Dashboard](https://developer.spotify.com/dashboard)
2. Click **"Create App"**
3. Fill in:
   - **App name:** `novatorem` (or anything)
   - **App description:** `GitHub profile widget`
   - **Redirect URI:** `http://localhost:8888/callback`
4. Click **"Save"**
5. Copy your **Client ID** and **Client Secret**

## 2. Get Refresh Token

### Option A: Using the Browser (Recommended)

1. Open this URL in your browser (replace `YOUR_CLIENT_ID`):

```
https://accounts.spotify.com/authorize?client_id=YOUR_CLIENT_ID&response_type=code&redirect_uri=http://localhost:8888/callback&scope=user-read-currently-playing%20user-read-recently-played
```

2. Authorize the app. You'll be redirected to `http://localhost:8888/callback?code=LONG_CODE_HERE`

3. Copy the `code` parameter from the URL

4. Run this curl command (replace placeholders):

```bash
curl -X POST "https://accounts.spotify.com/api/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code" \
  -d "code=YOUR_CODE_HERE" \
  -d "redirect_uri=http://localhost:8888/callback" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET"
```

5. From the JSON response, copy the `refresh_token` value

### Option B: Using the helper script

```bash
cd novatorem
cp .env.example .env
# Fill in SPOTIFY_CLIENT_ID and SPOTIFY_SECRET_ID

# Then use the built-in setup flow from SetUp.md
```

## 3. Deploy to Vercel

### Step-by-Step

1. **Push the novatorem repo to GitHub:**
   ```bash
   cd novatorem
   git add -A
   git commit -m "feat: custom spotify widget"
   git push origin main
   ```

2. **Go to [vercel.com](https://vercel.com)** and sign in with GitHub

3. **Import the `novatorem` repository**

4. **Set Environment Variables** in Vercel project settings:
   | Variable | Value |
   |----------|-------|
   | `SPOTIFY_CLIENT_ID` | Your Spotify Client ID |
   | `SPOTIFY_SECRET_ID` | Your Spotify Client Secret |
   | `SPOTIFY_REFRESH_TOKEN` | Your refresh token |

5. **Deploy** — Vercel will auto-detect the Python project

6. **Verify** by visiting: `https://novatorem-inzaghii17.vercel.app/api`

### Vercel Project Settings

- **Framework Preset:** Other
- **Build Command:** (leave empty)
- **Output Directory:** (leave empty)
- **Install Command:** `pip install -r requirements.txt`

> **Note:** If your Vercel URL is different from `novatorem-inzaghii17.vercel.app`,
> update the Spotify widget URL in your profile README accordingly.

## 5. Contribution Snake Setup

The snake animation is generated by a GitHub Action.

### Enable the Workflow

1. Go to your profile repo on GitHub
2. Click **"Actions"** tab
3. Click **"I understand my workflows, go ahead and enable them"**
4. Find **"Generate Snake Animation"**
5. Click **"Run workflow"** → **"Run workflow"** (first manual run)

The workflow will:
- Run daily at 00:00 UTC
- Generate `github-snake.svg` and `github-snake-dark.svg`
- Push them to the `output` branch

### Permissions

Make sure your repo has:
- **Settings → Actions → General → Workflow permissions:** "Read and write permissions"

## 7. Troubleshooting

### Widget shows error SVG
- Check Vercel environment variables are set correctly
- Verify refresh token is valid (tokens don't expire but can be revoked)
- Check Vercel function logs: Vercel Dashboard → Project → Functions → Logs

### Widget shows "No tracks available"
- Play a song on Spotify first (widget needs at least one recent track)
- Make sure you authorized the correct Spotify account

### Snake SVG not appearing
- Run the workflow manually first
- Check that the `output` branch was created
- Verify Actions have write permissions

### Stats cards not loading
- GitHub readme stats can be slow; give it a minute
- If persistent, try deploying your own instance of github-readme-stats

### SVG animations not working
- GitHub strips `<script>` tags but allows CSS animations
- All animations in this project use CSS `@keyframes` which GitHub supports
- Some SVG animations may not render in GitHub's dark mode preview — refresh the page
